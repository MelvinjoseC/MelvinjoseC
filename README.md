### About Me

I'm a full-stack developer with a Python and Django foundation who's actively moving toward **SRE, Platform Engineering, and DevOps**. My interest isn't just shipping features — it's making sure systems stay observable, automated, and resilient once they're live. I'm drawn to the boundary between application code and the infrastructure it runs on: CI/CD pipelines, containerization, deployment automation, and the tooling that lets teams ship confidently instead of nervously.

- 🔧 Comfortable with: **Python, Django, Docker, NGINX, Jenkins, GitHub Actions, AWS (EC2/RDS)**
- 📚 Currently leveling up: **Kubernetes, Terraform, Prometheus/Grafana, infrastructure as code**
- 🎯 Goal: bridge application development and platform reliability — writing code that's also easy to operate
- 🗣️ Always happy to talk about CI/CD design, deployment automation, or Python tooling for ops

### What I Actually Work On

- **CI/CD pipelines** — building and maintaining Jenkins and GitHub Actions workflows that take code from commit to deployment with minimal manual steps.
- **Containerized deployments** — Dockerizing Django applications and serving them through NGINX, with an eye toward making the setup portable across environments.
- **Cloud infrastructure on AWS** — deploying and managing services on EC2, with RDS for managed database needs.
- **Automation tooling** — writing Python scripts and Tkinter-based utilities to remove repetitive manual work from engineering workflows, including Autodesk Inventor automation.
- **Database performance** — tuning and optimizing MySQL, PostgreSQL, and MSSQL for applications under real load.
